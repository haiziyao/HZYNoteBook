# /connect — 知识连接

深度分析一篇笔记的内容，在整个知识库中搜索与之相关的笔记，发现隐藏的关联，帮助用户编织知识网络。

## 输入

`$ARGUMENTS` 是一个笔记的文件路径。如果为空，使用当前编辑器中打开的笔记。

## 执行步骤

### 第一步：读取目标笔记

1. 读取目标笔记的完整内容
2. 如果路径为空，从编辑器上下文获取当前笔记
3. 记录笔记标题和所在文件夹

### 第二步：提取关键概念

从笔记内容中提取以下元素：
- 核心主题（1-3 个）
- 关键概念和术语（3-7 个）
- 提到的人名、公司、工具、技术等实体
- 核心论点或观点

### 第三步：全库搜索

1. 对每个关键概念和主题，使用 Grep 工具搜索整个 vault
2. 排除目标笔记自身
3. 收集所有匹配的笔记路径
4. 去重后，选取匹配度最高的笔记（最多 10 篇）
5. 读取这些笔记的完整内容

### 第四步：分析语义关联

对每篇候选笔记进行深度分析，判断关联类型：

- **强关联**：内容直接讨论相同主题，观点互相印证或对比，属于同一知识体系
- **弱关联**：主题相近但角度不同，可能带来跨领域启发，共享某些底层概念
- **无实际关联**：仅关键词碰巧匹配，内容实际无关 → 排除

### 第五步：识别知识空白

分析目标笔记涉及的主题网络：
- 用户在 A 和 C 主题上有笔记，但缺少连接 A 和 C 的 B 主题
- 某个论点缺少支撑数据或案例
- 某个领域只有浅层记录，缺少深度分析

### 第六步：生成关联地图

```markdown
## 🔗 关联地图：{笔记标题}

### 强关联（内容直接相关）
- [[笔记A]] — {具体关联理由，说明两篇笔记的内容如何相关}
- [[笔记B]] — {具体关联理由}

### 弱关联（主题相近，可能有启发）
- [[笔记C]] — {关联理由，说明可能带来什么启发}
- [[笔记D]] — {关联理由}

### 知识空白
- {描述缺失的知识环节，建议用户补充什么内容}
- {描述另一个知识空白}
```

### 第七步：确认添加链接

向用户询问："是否将以上关联以 wikilink 形式添加到笔记末尾？"

如果用户确认：
1. 在目标笔记末尾添加 `## 相关笔记` 部分
2. 列出所有强关联和弱关联的 wikilink
3. 保存笔记
