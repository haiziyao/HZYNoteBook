# /ask — 知识库问答

用户提出问题时，优先从 vault 中的笔记寻找答案。用户自己积累的知识比通用知识更有价值，回答时必须优先引用 vault 内容。

## 输入

`$ARGUMENTS` 是用户用自然语言提出的问题。

## 执行步骤

### 第一步：解析问题

分析用户的问题，提取：
- 核心问题是什么
- 涉及哪些关键词和主题
- 生成 3-5 组搜索关键词（包含同义词和相关词）

### 第二步：搜索知识库

1. 使用 Grep 工具对每组关键词搜索整个 vault
2. 收集所有匹配的笔记路径
3. 去重后按匹配度排序
4. 读取排名最高的笔记（最多 15 篇）

### 第三步：评估覆盖度

判断 vault 中的内容对问题的覆盖程度：

- **完全覆盖**：vault 中有足够的笔记可以全面回答问题
- **部分覆盖**：vault 中有相关内容但不完整
- **无覆盖**：vault 中没有与问题相关的内容

### 第四步：生成回答

**完全覆盖时：**

综合 vault 中的笔记内容生成回答。要求：
- 以"你的笔记中记录了..."、"根据你在 [[笔记名]] 中的记录..."等方式引用
- 每个引用的观点都标注来源 wikilink
- 如果多篇笔记有相关内容，进行综合分析
- 明确区分"你的笔记内容"和"我的分析补充"

```markdown
## 回答

{综合 vault 笔记内容的回答，每处引用标注 [[来源笔记]]}

## 引用来源
- [[笔记A]] — {引用了其中的哪个观点或信息}
- [[笔记B]] — {引用了其中的哪个观点或信息}
```

**部分覆盖时：**

先回答 vault 中有的部分，然后指出缺失：

```markdown
## 回答

{基于 vault 笔记的回答}

## 引用来源
- [[笔记A]] — {引用内容}

## 知识库中暂缺的部分
{指出问题中哪些方面在 vault 中没有找到相关记录}

需要我联网搜索补充吗？如果找到有价值的信息，可以用 /capture 存入知识库。
```

**无覆盖时：**

```markdown
你的知识库中暂无与"{问题关键词}"相关的内容。

需要我联网搜索吗？如果找到有价值的信息，可以用 /capture 存入知识库，方便下次查阅。
```
