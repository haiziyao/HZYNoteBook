# /report — 工作报告生成

扫描用户的每日工作项和工作专项文件夹，提取已完成事项和关键产出，自动生成结构化的周报或月报。

## 输入

`$ARGUMENTS` 可以包含：
- 报告类型：周报（默认）或 月报
- 时间范围：本周、上周、本月、上月等

## 执行步骤

### 第一步：确定报告参数

解析输入，确定：
- 报告类型：周报或月报
- 日期范围：
  - 周报默认：本周一到今天（如果是周一，则取上周）
  - 月报默认：本月1号到今天
  - 用户指定"上周"：上周一到上周日
  - 用户指定"上月"：上月1号到上月最后一天

### 第二步：扫描每日工作项

1. 扫描 `📅 日常工作/每日工作项/` 文件夹
2. 筛选日期范围内的文件（根据文件名中的日期）
3. 读取每个文件，提取：
   - 已完成事项：标记为 `- [x]`、`✅`、`DONE` 的条目
   - 未完成事项：标记为 `- [ ]` 的条目
   - 备注内容

### 第三步：扫描工作专项

1. 扫描 `💼 工作专项/` 文件夹下所有子文件夹
2. 筛选在日期范围内被修改的文件
3. 这些文件代表具体的工作产出（文档、方案、报告等）

### 第四步：分类汇总

将所有已完成事项按工作板块分类：
- 根据每日工作项中的分组标题
- 根据工作专项的子文件夹名称
- 如果无法分类，归入"其他"

### 第五步：提取下期计划

1. 从 `📅 日常工作/近期规划/` 中提取与下一周期相关的计划
2. 从最近的每日工作项中提取未完成的待办事项

### 第六步：生成报告

```markdown
# {周报/月报}：{YYYY-MM-DD} ~ {YYYY-MM-DD}

## 本{周/月}完成事项

### {工作板块A}
- {完成项1}
- {完成项2}

### {工作板块B}
- {完成项1}

## 关键产出
- [[文件路径1]] — {简要描述该产出}
- [[文件路径2]] — {简要描述该产出}

## 下{周/月}计划
- {计划事项1}
- {计划事项2}

## 工作量统计
- 总完成事项：{X} 项
- 活跃工作板块：{板块A}、{板块B}、{板块C}
- 关键产出文档：{Y} 份
```

### 第七步：保存报告

根据报告类型保存到对应位置：
- 周报 → `📅 日常工作/工作报告/周报/周报-{YYYY-MM-DD}.md`（日期为该周周一）
- 月报 → `📅 日常工作/工作报告/月报/月报-{YYYY-MM}.md`

同时在聊天中展示完整报告内容。
