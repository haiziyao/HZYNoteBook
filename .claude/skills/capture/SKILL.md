# /capture — 万能捕获

将用户提供的任何信息转化为结构化笔记，存入知识库的合适位置。

## 输入

`$ARGUMENTS` 可以是以下三种之一：
- 一段文字内容
- 一个 URL 链接
- 空（处理当前正在编辑的笔记）

## 执行步骤

### 第一步：识别输入类型

判断 `$ARGUMENTS` 属于哪种类型：
- 如果以 `http://` 或 `https://` 开头 → 类型为 URL
- 如果有实际文字内容 → 类型为文本
- 如果为空 → 类型为当前笔记

### 第二步：提取内容

**URL 类型：**
1. 使用 WebFetch 工具获取页面内容
2. 提取页面标题作为笔记标题
3. 提取关键观点（3-5 个要点）
4. 生成一段摘要（100-200 字）
5. 记录原始 URL 作为 source

**文本类型：**
1. 分析文本内容，识别核心主题
2. 根据内容生成简洁的笔记标题
3. 整理内容结构（分段、提炼要点）

**当前笔记类型：**
1. 从编辑器上下文读取当前打开的笔记
2. 分析现有内容，增强结构（补充 frontmatter、分段、提炼要点）

### 第三步：生成结构化笔记

使用以下格式生成笔记：

```yaml
---
title: {笔记标题}
date: {今天的日期，YYYY-MM-DD}
tags: [{根据内容自动生成2-5个标签}]
source: {如果有来源URL或描述则填写}
---
```

笔记正文结构：
- 如果是 URL 来源：`## 摘要` → `## 关键要点` → `## 原文要点`
- 如果是文本内容：根据内容自然分段，添加合适的标题层级
- 如果是增强现有笔记：保留原有内容，补充结构

### 第四步：发现关联笔记

1. 从笔记内容中提取 3-5 个关键词
2. 使用 Grep 工具在 vault 中搜索每个关键词
3. 找到相关笔记后，在笔记末尾添加：

```markdown
## 相关笔记
- [[相关笔记A]]
- [[相关笔记B]]
```

### 第五步：确定存储位置

根据笔记内容判断应该存入哪个文件夹：
- 与用户具体工作项目相关 → `💼 工作专项/` 下的对应子文件夹
- 属于某个专业知识领域 → `📚 专业领域/` 下的对应子文件夹
- 属于个人观点、反思、感悟 → `💡 个人思考/`
- 与 AI 工具或技术相关 → `🤖 AI 学习&实操/`
- 无法明确分类 → `📝 日常记录/`

### 第六步：确认并保存

1. 向用户展示生成的笔记预览（标题、标签、目标文件夹）
2. 询问用户："笔记将保存到 `{目标文件夹}/{标题}.md`，确认？还是需要调整？"
3. 用户确认后保存笔记
