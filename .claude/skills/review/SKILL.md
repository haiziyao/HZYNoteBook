# /review — 定期回顾

总结用户在指定时间范围内的知识积累情况，按领域分类汇总，发现跨领域的洞察，提出深度思考问题帮助用户内化知识。

## 输入

`$ARGUMENTS` 可以包含：
- 时间范围：本周、上周、本月、上月、最近N天（默认：最近7天）
- 文件夹路径（可选）：指定只回顾某个文件夹

## 执行步骤

### 第一步：确定时间范围

解析用户输入的时间范围：
- "本周" → 本周一到今天
- "上周" → 上周一到上周日
- "本月" → 本月1号到今天
- "上月" → 上月1号到上月最后一天
- "最近N天" → 从今天往前推 N 天
- 默认 → 最近 7 天

### 第二步：扫描笔记

1. 使用 Glob 工具扫描指定范围（整个 vault 或指定文件夹）的所有 `.md` 文件
2. 对每个文件检查修改日期或 frontmatter 中的 date 字段
3. 筛选出在时间范围内创建或修改的笔记
4. 统计总数

### 第三步：阅读并分类

读取每篇符合条件的笔记，按所在文件夹分类：

- `📚 专业领域/` 下的笔记 → 归入"学习收获"
- `💡 个人思考/` 下的笔记 → 归入"个人思考"
- `💼 工作专项/` 下的笔记 → 归入"工作产出"
- `🤖 AI 学习&实操/` 下的笔记 → 归入"AI 进展"
- `📅 日常工作/` 下的笔记 → 归入"日常工作"
- `📝 日常记录/` 下的笔记 → 归入"待整理"

### 第四步：分析跨领域关联

审视不同领域的笔记内容，寻找：
- 不同领域笔记之间的共同主题
- 某个领域的思考是否能启发另一个领域
- 是否有反复出现的关键词或概念

### 第五步：生成回顾报告

```markdown
## 📖 知识回顾：{时间范围}

### 本期概览
- 新增/更新笔记：{X} 篇
- 涉及领域：{领域A}、{领域B}、{领域C}

### 学习收获
{按 📚 专业领域 分组总结，每个子领域列出学到的关键知识点}

### 工作产出
{按 💼 工作专项 分组总结，列出主要产出}

### 个人思考
{总结 💡 个人思考 中的主要思考主题和观点演变}

### AI 进展
{总结 🤖 AI 学习&实操 中的工具使用和学习进展}

### 跨领域洞察
{分析不同领域笔记之间的有趣关联，指出潜在的知识交叉点}

### 思考问题
1. {基于本期学习内容提出的深度问题，帮助用户进一步思考}
2. {另一个思考问题}
3. {另一个思考问题}

### 待整理
- 📝 日常记录中有 {n} 篇笔记尚未归类，建议使用 /organize 整理
```

### 第六步：展示报告

将回顾报告直接展示在聊天中。不要自动保存，除非用户要求保存。

如果用户要求保存，保存到 `📅 日常工作/工作报告/` 下，文件名格式：`回顾-{时间范围}-{日期}.md`
