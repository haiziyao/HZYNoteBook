# /daily — 每日启动

每天工作开始时运行，自动创建今日工作项文件，迁移昨日未完成待办，快速回顾最近的笔记动态。

## 输入

无需参数。

## 执行步骤

### 第一步：获取日期信息

1. 获取今天的日期（YYYY-MM-DD 格式和 YYYYMMDD 格式）
2. 计算昨天的日期

### 第二步：检查今日文件

检查今日工作项文件是否已存在：`📅 日常工作/每日工作项/{YYYYMMDD}.md`

- 如果已存在 → 读取内容，跳到第五步
- 如果不存在 → 继续第三步创建

### 第三步：读取昨日文件

检查昨日工作项文件：`📅 日常工作/每日工作项/{昨日YYYYMMDD}.md`

如果存在：
1. 读取全部内容
2. 提取所有未完成事项（`- [ ]` 格式的条目）
3. 提取昨日已完成事项摘要

如果不存在：
- 记录"昨日无工作项文件"

### 第四步：创建今日工作项文件

使用以下模板创建 `📅 日常工作/每日工作项/{YYYYMMDD}.md`：

```markdown
# {YYYY-MM-DD} DONE & TODO

## 昨日回顾
{如果有昨日文件，列出昨日已完成事项摘要}
{如果没有昨日文件，写"无昨日记录"}

## 今日待办
{迁移昨日未完成事项，保持 - [ ] 格式}
- [ ]

## 今日完成
-

## 备注

```

### 第五步：扫描最近笔记动态

1. 使用 Glob 扫描整个 vault 的 `.md` 文件
2. 筛选最近 3 天内创建或修改的笔记
3. 排除 `📅 日常工作/每日工作项/` 下的文件（避免循环引用）
4. 按修改时间排序，取最近 10 篇

### 第六步：展示每日简报

在聊天中展示以下内容：

```markdown
☀️ 早上好！今天是 {YYYY-MM-DD}

📋 昨日未完成（已迁移到今日待办）：
- {未完成事项1}
- {未完成事项2}
（如果没有未完成事项，显示"昨日事项已全部完成！"）

📝 最近更新的笔记：
- [[笔记标题1]]（{昨天/前天/3天前}）
- [[笔记标题2]]（{时间}）
- ...

今日工作项已创建：[[📅 日常工作/每日工作项/{YYYYMMDD}.md]]
```
